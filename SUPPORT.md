# Support

Use the right channel so the issue tracker stays focused:

- Bug reports: open a GitHub Issue with the bug report form.
- Feature requests: open a GitHub Issue with the feature request form.
- Architecture changes: open an RFC issue before implementation.
- Usage questions and troubleshooting: use GitHub Discussions once enabled.
- Security concerns: follow `SECURITY.md` and avoid public issues.

If you are not sure whether something is a bug or a support request, start by describing the expected behavior, the actual behavior, and the reproduction steps.
